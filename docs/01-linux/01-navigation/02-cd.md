# cd Command Documentation

## Name and Synopsis

The `cd` utility is used to change the working directory of the current shell execution environment.

**Basic Usage:**

```bash
cd [-L|-P] [directory]
cd -
```

---

## Description

The `cd` utility performs a sequence of steps to update the current working directory:

1. **No Directory Specified:**
    * If the `HOME` environment variable is empty or undefined, the behavior is implementation-defined and no changes are made.
    * If `HOME` is set to a non-empty value, `cd` behaves as if that directory was specified as the operand.
2. **Path Resolution:**
    * If the operand starts with a `/`, it is treated as an absolute path [4].
    * If not, `cd` searches through the `CDPATH` environment variable. It tests each pathname in `CDPATH` concatenated with the directory operand to see if it names an existing directory.
3. **Canonicalization:** The path is converted to its canonical form by deleting dot components and resolving dot-dot (`..`) components.
4. **Final Action:** The utility performs actions equivalent to the `chdir()` function. If successful, the `PWD` environment variable is updated, and the `OLDPWD` variable is set to the previous working directory.

---

## Options Reference

The following options are available for the `cd` command:

| Option | Description | Detailed Explanation |
| :--- | :--- | :--- |
| **`-L`** | **Logical Dot-Dot** | Handles the `..` operand logically. Symbolic link components are not resolved before `..` components are processed. This is the default behavior if no option is specified [4]. |
| **`-P`** | **Physical Dot-Dot** | Handles the `..` operand physically. Symbolic link components are resolved before `..` components are processed [4]. |

*Note: If both `-L` and `-P` are specified, the last option provided is used.*

---

## Operands

The following operands are supported by the command [4]:

| Operand | Description | Equivalent Command |
| :--- | :--- | :--- |
| **`directory`** | An absolute or relative pathname of the directory to become the new working directory. | `cd /path/to/dir` |
| **`-`** | Changes to the previous working directory and writes its name to standard output. | `cd "$OLDPWD" && pwd` |

---

## Environment Variables

Several environment variables affect the execution of `cd` [4]:

* **`CDPATH`**: A colon-separated list of pathnames used to search for directories when a relative path is provided.
* **`HOME`**: The name of the directory used when no directory operand is specified.
* **`OLDPWD`**: Stores the pathname of the previous working directory (used by `cd -`).
* **`PWD`**: The current working directory; updated by the `cd` utility.

---

## Exit Status and Error Handling

* **Success:** Returns 0 if the directory was successfully changed.
* **Failure:** Returns a value greater than 0 if an error occurred.
* **Consequences of Errors:** If an error occurs during execution, the working directory remains unchanged.

---

## Application Usage Note

Because `cd` affects the current shell environment, it is almost always provided as a **shell regular built-in**.

If you call `cd` in a subshell or as a separate utility (for example, using `find . -exec cd {} \;`), it will change the directory for that subshell only and **will not affect the working directory of the caller's environment**.
