# cp Command Documentation

## Name and Synopsis

The `cp` command is used to copy files and directories.

**Basic Usage:**

```bash
cp [OPTION]... [-T] SOURCE DEST
cp [OPTION]... SOURCE... DIRECTORY
cp [OPTION]... -t DIRECTORY SOURCE...
```

---

## Description

The `cp` command copies the specified `SOURCE` to `DEST`, or multiple `SOURCE(s)` into a `DIRECTORY`.

Mandatory arguments for long options are also mandatory for short options.

---

## Options Reference

The following options are available for the `cp` command [10]:

| Option | Long Form | Description |
| :--- | :--- | :--- |
| **`-a`** | **`--archive`** | Same as `-dR --preserve=all`. |
| **`--attributes-only`** | N/A | Do not copy the file data, just the attributes. |
| **`--backup[=CONTROL]`** | N/A | Make a backup of each existing destination file. |
| **`-b`** | N/A | Like `--backup` but does not accept an argument. |
| **`--copy-contents`** | N/A | Copy contents of special files when recursive. |
| **`-d`** | N/A | Same as `--no-dereference --preserve=links`. |
| **`--debug`** | N/A | Explain how a file is copied; implies `-v`. |
| **`-f`** | **`--force`** | If an existing destination file cannot be opened, remove it and try again (ignored when `-n` is used) [10]. |
| **`-i`** | **`--interactive`** | Prompt before overwrite (overrides a previous `-n` option). |
| **`-n`** | **`--no-clobber`** | (Deprecated) Silently skip existing files. See also `--update`. |
| **`--no-copy`** | N/A | Do not copy if renaming fails. |
| **`--strip-trailing-slashes`** | N/A | Remove any trailing slashes from each `SOURCE` argument. |
| **`-S`** | **`--suffix=SUFFIX`** | Override the usual backup suffix. |
| **`-t`** | **`--target-directory=DIRECTORY`** | Copy all `SOURCE` arguments into `DIRECTORY`. |
| **`-T`** | **`--no-target-directory`** | Treat `DEST` as a normal file. |
| **`--update[=UPDATE]`** | N/A | Control which existing files are updated; `UPDATE={all,none,none-fail,older(default)}`. |
| **`-u`** | N/A | Equivalent to `--update[=older]`. |
| **`-v`** | **`--verbose`** | Explain what is being done. |
| **`-x`** | **`--one-file-system`** | Stay on this file system. |
| **`-Z`** | N/A | Set SELinux security context of destination file to default type. |
| **`--context[=CTX]`** | N/A | Like `-Z`, or if `CTX` is specified, set the SELinux or SMACK security context to `CTX`. |
| **`--help`** | N/A | Display help message and exit. |
| **`--version`** | N/A | Output version information and exit. |

---

## Update Behavior

When using the `--update` option, the following controls apply to existing files in the destination:

* **`all`**: The default operation when an `--update` option is *not* specified; results in all existing files in the destination being replaced.
* **`none`**: Like the `--no-clobber` option, no files in the destination are replaced, and skipped files do not induce a failure.
* **`none-fail`**: Ensures no files are replaced in the destination, but any skipped files are diagnosed and induce a failure.
* **`older`**: The default operation when `--update` is specified; results in files being replaced if they're older than the corresponding source file.

---

## Backup Behavior

The backup suffix is typically `~`, unless set with `--suffix` or `SIMPLE_BACKUP_SUFFIX`. The version control method may be selected via the `--backup` option or through the `VERSION_CONTROL` environment variable. Options include:

* **none, off**: Never make backups (even if `--backup` is given).
* **numbered, t**: Make numbered backups.
* **existing, nil**: Numbered if numbered backups exist, simple otherwise.
* **simple, never**: Always make simple backups.
