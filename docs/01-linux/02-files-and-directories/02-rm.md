# rm Command Documentation

## Name and Synopsis

The `rm` command is used to remove files or directories.

**Basic Usage:**

```bash
rm [OPTION]... [FILE]...
```

---

## Description

The `rm` command removes each specified file. By default, it does not remove directories; the `--recursive` (`-r` or `-R`) option must be used to remove a directory and its contents.

The command includes interactive prompts to prevent accidental deletions:

* If `-I` or `--interactive=once` is provided, `rm` prompts before proceeding with more than three files or a recursive operation.
* Otherwise, if a file is unwritable, standard input is a terminal, and the `-f` option is not given (or if `-i`/`--interactive=always` is specified), `rm` prompts the user before removing each file.

---

## Options Reference

The following options are available for the `rm` command:

| Option | Long Form | Description |
| :--- | :--- | :--- |
| **`-f`** | `--force` | Ignore nonexistent files and arguments; never prompt. |
| **`-i`** | N/A | Prompt the user before every removal. |
| **`-I`** | N/A | Prompt once before removing more than three files or when removing recursively. |
| **`--interactive[=WHEN]`** | N/A | Prompt according to `WHEN`: `never`, `once` (-I), or `always` (-i); defaults to always if no value is given. |
| **`--one-file-system`** | N/A | When removing recursively, skip directories on a file system different from the command line argument. |
| **`--no-preserve-root`** | N/A | Do not treat the root directory (`/`) specially. |
| **`--preserve-root[=all]`** | N/A | Do not remove `/` (default); with `all`, reject arguments on a separate device from its parent. |
| **`-r`, `-R`** | `--recursive` | Remove directories and their contents recursively. |
| **`-d`** | `--dir` | Remove empty directories. |
| **`-v`** | `--verbose` | Explain what is being done during the operation. |
| **`--help`** | N/A | Display help message and exit [7]. |
| **`--version`** | N/A | Output version information and exit [7]. |

---

## Important Notes

* **Safety Protections:** Any attempt to remove a file whose last name component is `.` or `..` is rejected with a diagnostic.
* **Filenames Starting with Hyphens:** To remove a file starting with a hyphen (e.g., `-foo`), use either `rm -- -foo` or `rm ./-foo` to prevent the command from interpreting the filename as an option.
* **Data Recovery:** It may be possible to recover some contents of a removed file with sufficient expertise. For greater assurance that contents are unrecoverable, consider using `shred(1)`.
