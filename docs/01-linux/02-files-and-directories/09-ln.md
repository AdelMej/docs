# ln Command Documentation

## Name and Synopsis

The `ln` command is used to make links between files.

```bash
ln [OPTION]... [-T] TARGET LINK_NAME
ln [OPTION]... TARGET
ln [OPTION]... TARGET... DIRECTORY
ln [OPTION]... -t DIRECTORY TARGET...
```

---

## Description

The `ln` command provides four different ways to create links. In the first form, it creates a link to a `TARGET` named `LINK_NAME`. In the second form, it creates a link to a `TARGET` in the current directory. The third and fourth forms allow for creating multiple links for various `TARGET`s within a specified `DIRECTORY`.

By default, the command creates hard links, while the `--symbolic` (`-s`) option is used to create symbolic links. Generally, the destination name should not already exist, and when creating hard links, the `TARGET` must exist. Symbolic links can hold arbitrary text; if they are resolved later, a relative link is interpreted in relation to its parent directory. Mandatory arguments for long options are also mandatory for short options.

---

## Options Reference

| Option | Long Form | Description |
| :--- | :--- | :--- |
| -b | --backup[=CONTROL] | Make a backup of each existing destination file |
| -d | -F, --directory | Allow the superuser to attempt to hard link directories, if supported by the system |
| -f | --force | Remove existing destination files |
| -i | --interactive | Prompt whether to remove destinations |
| -L | --logical | Dereference `TARGET`s that are symbolic links |
| -n | --no-dereference | Treat `LINK_NAME` as a normal file if it is a symbolic link to a directory |
| -P | --physical | Make hard links directly to symbolic links |
| -r | --relative | With `-s`, create links relative to link location |
| -s | --symbolic | Make symbolic links instead of hard links |
| -S | --suffix=SUFFIX | Override the usual backup suffix (default is '~') |
| -t | --target-directory=DIRECTORY | Specify the `DIRECTORY` in which to create the links |
| -T | --no-target-directory | Treat `LINK_NAME` as a normal file always |
| -v | --verbose | Print the name of each linked file |
| --help | | Display this help and exit |
| --version | | Output version information and exit |

---

## Configuration / Files Modified (if applicable)

General behavior: This command modifies the filesystem by creating new entries (links). It does not modify specific system configuration files.

---

## Examples

* **Create a symbolic link:**

```bash
 ln -s /path/to/original_file link_name
```

* **Create a hard link:**

```bash
ln original_file hard_link_name
```

* **Create multiple symbolic links in a specific directory:**

```bash
ln -s -t /path/to/destination/ folder1 folder2
```

* **Force overwrite an existing link:**

```bash
ln -sf /path/to/original_file existing_link_name
```

---

## Exit Values

* 0: Success.
* Non-zero: Indicates an error occurred (e.g., file not found, permission denied).

---

## Caveats / Notes

* **Hard Links:** Each `TARGET` must exist when creating hard links.
* **Symbolic Links:** These can hold arbitrary text; if resolved, a relative link is interpreted in relation to its parent directory.
* **Default Behavior:** By default, the command creates hard links and will fail if the destination name already exists.
