# ls Command Documentation

## Name and Synopsis

The `ls` command is used to list directory contents.

**Basic Usage:**

```bash
ls [OPTION]... [FILE]...
```

---

## Description

By default, `ls` lists information about the files in the current directory. If no specific sorting options are provided (such as `-c`, `-f`, `-t`, `-u`, `-v`, `-S`, `-X`, or `--sort`), entries are sorted alphabetically.

The command accepts mandatory arguments for long options, which must also be provided if using their corresponding short options.

---

## Options Reference

The following options are available for the `ls` command:

| Option | Description | Detailed Explanation |
| :--- | :--- | :--- |
| **`-a`**, `--all` | **Show All Entries** | Do not ignore entries starting with a dot (e.g., hidden files). |
| **`-A`**, `--almost-all` | **Almost All Entries** | Do not list implied `.` and `..` entries. |
| **`-b`**, `--escape` | **C-style Escapes** | Print C-style escapes for non-graphic characters. |
| **`-c`** | **Sort by ctime** | With `-lt`: sort by and show `ctime` (last change of file status). With `-l`: show `ctime` and sort by name. Otherwise: sort by `ctime`, newest first. |
| **`-C`** | **Columns** | List entries by columns. |
| **`-d`**, `--directory` | **List Directories** | List the directories themselves, not their contents. |
| **`-f`** | **No Sort** | Same as `-a -U`. |
| **`-F`**, `--classify` | **Append Indicator** | Append an indicator (one of `*/=>@|`) to entries. |
| **`-g`** | **No Owner** | Like `-l`, but do not list the owner. |
| **`-G`**, `--no-group` | **No Group** | In a long listing, do not print group names. |
| **`-h`**, `--human-readable` | **Human Readable** | With `-l` and `-s`, print sizes like `1K`, `234M`, `2G`, etc. |
| **`-H`**, `--dereference-command-line` | **Follow Links** | Follow symbolic links listed on the command line. |
| **`-i`**, `--inode` | **Show Inode** | Print the index number (inode) of each file. |
| **`-I`**, `--ignore=PATTERN` | **Ignore Pattern** | Do not list implied entries matching a shell PATTERN. |
| **`-k`**, `--kibibytes` | **KB Blocks** | Default to 1024-byte blocks for file system usage; used only with `-s` and per directory totals. |
| **`-l`** | **Long Listing** | Use a long listing format (showing permissions, owner, size, date, etc.). |
| **`-L`**, `--dereference` | **Dereference Links** | When showing information for a symbolic link, show information for the file the link references rather than the link itself. |
| **`-m`** | **Comma Separated** | Fill width with a comma-separated list of entries. |
| **`-n`**, `--numeric-uid-gid` | **Numeric IDs** | Like `-l`, but list numeric user and group IDs instead of names. |
| **`-N`**, `--literal` | **Literal Names** | Print entry names without quoting. |
| **`-o`** | **No Group Info** | Like `-l`, but do not list group information. |
| **`-p`**, `--indicator-style=slash` | **Directory Indicator** | Append a `/` indicator to directories. |
| **`-q`**, `--hide-control-chars` | **Hide Characters** | Print `?` instead of non-graphic characters. |
| **`-Q`**, `--quote-name` | **Quote Names** | Enclose entry names in double quotes. |
| **`-r`**, `--reverse` | **Reverse Sort** | Reverse the order while sorting. |
| **`-R`**, `--recursive` | **Recursive List** | List subdirectories recursively. |
| **`-s`**, `--size` | **Show Size** | Print the allocated size of each file in blocks. |
| **`-S`** | **Sort by Size** | Sort by file size, largest first. |
| **`-t`** | **Sort by Time** | Sort by time, newest first (see `--time`). |
| **`-T`**, `--tabsize=COLS` | **Tab Stops** | Assume tab stops at each `COLS` instead of 8. |
| **`-u`** | **Sort by Access Time** | With `-lt`: sort by and show access time. With `-l`: show access time and sort by name. Otherwise: sort by access time, newest first. |
| **`-U`** | **No Sort** | Do not sort directory entries. |
| **`-v`** | **Natural Sort** | Natural sort of (version) numbers within text. |
| **`-w`**, `--width=COLS` | **Set Width** | Set output width to `COLS`. 0 means no limit. |
| **`-x`** | **Lines** | List entries by lines instead of by columns. |
| **`-X`** | **Extension Sort** | Sort alphabetically by entry extension. |
| **`-Z`**, `--context` | **Security Context** | Print any security context of each file. |
| **`-1`** | **One Per Line** | List one file per line. |

---

## Time and Size Information

* **Size Units:** The `SIZE` argument is an integer and optional unit (e.g., `10K` is 10*1024). Units are `K,M,G,T,P,E,Z,Y,R,Q` (powers of 1024) or `KB,MB,...` (powers of 1000).
* **Time Style:** With `-l`, `--time-style` can be `full-iso`, `long-iso`, `iso`, `locale`, or `+FORMAT`.

---

## Exit Status

* **0**: Success
* **1**: Minor problems (e.g., cannot access subdirectory)
* **2**: Serious trouble (e.g., cannot access command-line argument)
