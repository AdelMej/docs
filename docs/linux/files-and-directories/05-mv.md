# mv Command Documentation

## Name and Synopsis

The `mv` command is used to move or rename files.

**Basic Usage:**

```bash
mv [OPTION]... [-T] SOURCE DEST
mv [OPTION]... SOURCE... DIRECTORY
mv [OPTION]... -t DIRECTORY SOURCE...
```

---

## Description

The `mv` command renames a `SOURCE` to a `DEST`, or moves one or more `SOURCE(s)` into a `DIRECTORY`.

Mandatory arguments for long options are also required when using their corresponding short options.

---

## Options Reference

The following options are available for the `mv` command:

| Option | Long Form | Description |
| :--- | :--- | :--- |
| **`-b`** | **`--backup[=CONTROL]`** | Make a backup of each existing destination file. |
| **`--debug`** | N/A | Explain how a file is copied; implies `-v`. |
| **`--exchange`** | N/A | Exchange source and destination. |
| **`-f`** | **`--force`** | Do not prompt before overwriting. |
| **`-i`** | **`--interactive`** | Prompt before overwrite. |
| **`-n`** | **`--no-clobber`** | Do not overwrite an existing file. |
| **`--no-copy`** | N/A | Do not copy if renaming fails. |
| **`--strip-trailing-slashes`** | N/A | Remove any trailing slashes from each `SOURCE` argument. |
| **`-S`** | **`--suffix=SUFFIX`** | Override the usual backup suffix. |
| **`-t`** | **`--target-directory=DIRECTORY`** | Move all `SOURCE` arguments into `DIRECTORY`. |
| **`-T`** | **`--no-target-directory`** | Treat `DEST` as a normal file. |
| **`--update[=UPDATE]`** | N/A | Control which existing files are updated (options: `all`, `none`, `none-fail`, `older` \[default\]). |
| **`-u`** | N/A | Equivalent to `--update[=older]`. |
| **`-v`** | **`--verbose`** | Explain what is being done. |
| **`-Z`** | **`--context`** | Set SELinux security context of destination file to default type. |
| **`--help`** | N/A | Display help message and exit. |
| **`--version`** | N/A | Output version information and exit. |

---

## Update Behavior

When using the `--update` option, the following controls apply to existing files in the destination:

* **`all`**: Default operation when `--update` is specified; replaces all existing files.
* **`none`**: Like `--no-clobber`; no files in the destination are replaced, and skipped files do not cause a failure.
* **`none-fail`**: No files are replaced, but any skipped files are diagnosed and induce a failure.
* **`older`**: Default for `--update`; files are replaced only if they are older than the corresponding source file.

---

## Backup Behavior

The backup suffix is typically `~` unless set otherwise with `--suffix` or the `SIMPLE_BACKUP_SUFFIX` environment variable. Version control methods include:

* **none, off**: Never make backups.
* **numbered, t**: Make numbered backups.
* **existing, nil**: Numbered if numbered backups exist, simple otherwise.
* **simple, never**: Always make simple backups.
