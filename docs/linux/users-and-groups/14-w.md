# w Command Documentation

## Name and Synopsis

The `w` command shows who is logged on and what they are doing.

```bash
w [option ...] [user]
```

---

## Description

The `w` command displays information about the users currently on the machine and their associated processes. The output header provides the current time, system uptime, the number of users currently logged on, and the system load averages for the past 1, 5, and 15 minutes.

For each user, the command displays the login name, the tty name, the remote host, login time, idle time, JCPU (CPU time used by all processes attached to the tty), PCPU (CPU time used by the current process listed in the "what" field), and the command line of their current process.

---

## Options Reference

| Option | Long Form | Description |
| :--- | :--- | :--- |
| `-c` | `--container` | Show the container uptime instead of system uptime in the header. |
| `-h` | `--no-header` | Do not print the header. |
| `-u` | `--no-current` | Ignores the username while figuring out the current process and CPU times. |
| `-s` | `--short` | Use the short format; does not print the login time, JCPU, or PCPU times. |
| `-t` | `--terminal` | Scan terminal devices (`/dev/tty*` and `/dev/pts/*`) to locate user sessions. |
| `-f` | `--from` | Toggle printing of the "from" (remote hostname) field. |
| `-i` | `--ip-addr` | Display IP address instead of hostname for the "from" field (works only on non-systemd platforms). |
| `-p` | `--pids` | Display PID of the login process and the "what" process in the "what" field. |
| `-o` | `--old-style` | Use old style output, printing blank space for idle times less than one minute. |
| `-V` | `--version` | Display version information. |
| `--help` | | Display help text and exit. |
| `user` | | Show information about the specified user only. |

## Configuration / Files Modified (if applicable)

The `w` command reads from the following files and is influenced by these environment variables:

**Files:**

* `/var/run/utmp`: Information about who is currently logged on (for non-systemd hosts).
* `/proc`: Process information.
* `/dev/tty*`, `/dev/pts/*`: Terminal device files scanned when using `--terminal` mode.

**Environment Variables:**

* `PROCPS_CONTAINER`: If set, `w` behaves as if the `--container` option was given.
* `PROCPS_USERLEN`: Overrides the default width of the username column (default is 8).
* `PROCPS_FROMLEN`: Overrides the default width of the "from" column (default is 16).

---

## Examples

* Display all logged-in users and their processes:

```bash
w
```

* Display information in short format:

```bash
w -s
```

* Show information for a specific user:

```bash
w username
```

* Display info without the header:

```bash
w -h
```

---

## Exit Values

* 0: Success.
* Non-zero: Indicates an error occurred (General behavior).

---

## Caveats / Notes

* **Terminal Mode Accuracy:** When using the `--terminal` option, `w` may not provide a true count of users; for example, a user with two xterms will show up twice, resulting in a different user count in the header.
* **Idle Time Issues:** Idle time for users on certain sessions, such as X/Wayland sessions, will be incorrect. The `--terminal` option provides correct idle times for terminal sessions on X/Wayland.
* **Systemd Restrictions:** On platforms using systemd(1), the `--ip-addr` option is ignored, and the "from" column will report the remote host as either a hostname or IP address.
* **Process Exclusion:** When using the `--terminal` option, `w` assumes processes with a parent PID of 0 or 1 are `agetty(8)` processes and will not display them; this is prone to both false positive and negative errors.
